## Working-Style Principles

2. **Migration discipline** — finished migrations, with the legacy code path removed (never just deprecated).
3. **Reliability mindset** — observable, reversible, gradient-rolled changes. Mitigate, clean up, then add monitoring.
4. **Communication as a deliverable** — documentation is a first-class artifact, not a footnote.
5. **Patient performance** — many small instrumented wins beat one heroic rewrite.
6. **Operator empathy** — CLIs, runbooks, and breakglass procedures for the engineer who comes next.
7. **Cross-language fluency** — coherent abstractions across language boundaries, with FFI plumbing in between.
8. **Multi-attempt resolve** — when a rollout has to revert, it gets re-attempted, not abandoned.
